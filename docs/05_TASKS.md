## Phase 0 — 외부 서비스 준비 [콘솔]

> 코드 전에 계정·키를 준비한다. 일부는 사업자등록(엔터랩스) 필요.

### T0.1 [콘솔] Supabase 프로젝트 생성
- 프로젝트 생성 → DB·API URL·anon key·service_role key 확보.
- **DoD**: 키 4종 확보, 빈 프로젝트 접속 가능.

### T0.2 [콘솔] GitHub 레포 + Vercel 연결
- 레포 생성 → Vercel 프로젝트 import → 자동 배포 파이프라인 확인.
- **DoD**: 빈 Next.js 앱이 Vercel URL로 뜸.

### T0.3 [콘솔] 소셜 로그인 앱 등록
- **구글**: GCP OAuth 동의화면 + 클라이언트 ID, Redirect URI(Supabase 콜백) 등록.
- **카카오**: 앱 생성 → 비즈앱 전환(사업자등록 필요) → (이름·연락처 수집 시) 카카오싱크 신청 → REST API Key를 Supabase에 등록.
- **네이버**: 개발자센터 앱 등록 → Callback URL 등록.
- Supabase Auth → Providers에서 3종 활성화.
- **DoD**: Supabase 대시보드에서 3 provider enabled.

### T0.4 [콘솔] 토스페이먼츠 가맹 + 키 + 자금 분리
- 가맹 신청(사업자) → 클라이언트/시크릿 키, webhook 시크릿 확보.
- **자금 분리 설계**: ① 에스크로 자금(파트너에게 갈 돈)은 토스/별도 계좌로 곁에 운영비와 분리. ② 곁에 책임 적립금용 별도 계좌 준비. (외부 보험 아닌 내부 적립, 성장기 외부 보증 전환 검토)
- **DoD**: 테스트 결제 키 확보, 자금 분리 계좌 구조 확정.

### T0.5 [콘솔] 도메인 연결
- yourside.cloud → Vercel DNS 연결, HTTPS 자동 발급 확인.
- **DoD**: yourside.cloud 접속 시 앱 표시.

### T0.6 [콘솔/법무] 법적 지위 확인 + 약관 준비
- 노무사·변호사에게 확인: ① 직업안정법 유료직업소개 신고 대상 여부, ② 파견법 적용 여부, ③ 집출 자문형 서비스의 도급 성격, ④ 통신판매중개업 신고.
- 약관·이용계약 초안 작성: 곁에=건별 용역 중개(도급 중개), 매칭피=중개 수수료, 결과물 책임=파트너, 근로·파견 관계 부존재 명시(TRD §7.5).
- **DoD**: 법적 지위 확인 완료, 약관 초안 확보. (오픈 전 필수, 코드와 병행 가능)

## Phase 3 — 사장님 흐름 [코드]

### T3.1 [코드] 일 맡기기
- 자유 서술 + 빠른 칩(집출 서비스 포함) → POST /api/requests.
- req_type은 비우고(관리자 분류), status=open.
- **DoD**: 의뢰 등록 → request 생성.

### T3.2 [코드] 의뢰 현황
- 4단계 진행바 + 24h 견적 기한 표시. GET /api/requests/:id.
- **DoD**: 등록 후 현황 화면에 상태 표시.

### T3.3 [코드] 견적 승인
- 익명 파트너 카드(연락처 비공개) + 곁에 평가 별점 + 총액(통합) + 견적서 PDF + 에스크로 안내.
- PATCH /api/deals/:id/approve.
- **DoD**: 견적 승인 → 결제 단계로.

### T3.4 [코드] 검수·정산
- 결과물 확인 + 검수완료/수정요청/문제신고 + 거래명세서 + 3일 자동확정.
- PATCH /api/deals/:id/confirm.
- **DoD**: 검수 완료 → deal.status=done.

## Phase 4 — 파트너 흐름 [코드]

### T4.1 [코드] 파트너 등록
- 분야 칩(집출 포함: 창업·경영 코칭 등) + 경력 구간 + 한줄 소개. "작업료 0%" 강조. 최소 입력.
- **DoD**: 등록 → partner 생성.

### T4.2 [코드] 의뢰 수락
- 곁에 책정 견적 + 작업 범위 + 납기 + 수락/조건상의/거절.
- PATCH /api/matching/:id.
- **DoD**: 수락 → deal 확정(견적 계산 반영).

### T4.3 [코드] 작업 현황 + 공통 5단계 워크플로우
- "작업료 안전 보관 중"(에스크로) + 받을 작업료(작업료만) + 결과물 업로드(Supabase Storage).
- **공통 5단계 워크플로우 체크리스트(수준 1 최소)**: deal 생성 시 deal_workflow 5단계 자동 생성. 파트너가 수집→구조화→생성→검증→납품을 'in_progress'→'done' 갱신. PATCH /api/deals/:id/workflow/:step.
- ⚠️ 도메인 구분 없는 공통 뼈대 1개. AI 도구 임베드·도메인 설정은 Phase 2(범위 밖).
- POST /api/deals/:id/submit (deliver 단계 후 결과물 제출).
- **DoD**: 5단계 체크리스트 표시·갱신, 결과물 업로드 → 사장님 검수 대기.

### T4.4 [코드] 정산·평가 완료
- 정산 확정 + 곁에 검토진 확인 안내 + 선택적 전할말 + 정산내역서.
- **DoD**: 정산 후 완료 화면.

## Phase 6 — 관리자 [코드]

### T6.1 [코드] 통합 대시보드
- 오늘 할 일 요약(매칭/진행/정산/문의 카운트) + 탭 전환.
- 매칭 대기: 키워드 추천 후보 → 파트너 직접 선택(POST /api/matching).
- 진행 중: 거래별 공통 5단계 워크플로우 진행 상태 확인.
- **DoD**: 새 의뢰가 매칭 대기에 뜸, 파트너 선택 가능, 진행 중 거래의 5단계 상태 조회.

### T6.2 [코드] 정산 실행 + 책임 적립금 (사람 버튼)
- POST /api/settlements/:id/release — **관리자만**. 감사 로그.
- 정산 시 곁에 책임 적립금 적립(원장 'accrue', append-only).
- POST /api/settlements/:id/refund — 사고·분쟁 시 적립금 충당('payout') + 사장님 환불.
- GET /api/admin/guarantee-fund — 책임 적립금 잔액·원장 조회.
- ⚠️ 에스크로 자금·운영비와 분리된 별도 계좌 기준으로 적립금 관리(잔액-원장 대사).
- **DoD**: 검수 완료 거래만 정산 실행 → released + 파트너 전달, 적립 기록. 환불 시 적립금 차감·원장 기록.

### T6.3 [코드] 곁에 검토 입력
- 별 5개 + 검토 의견(공개) + 사장님 협조도(내부 비공개). POST /api/reviews.
- **DoD**: 입력 → review(author_type=gyeotae) 저장, 파트너 카드에 "곁에 평가" 반영.

## 마일스톤 정리
| 단계 | 목표 |
|---|---|
| Phase 0~2 | 로그인되는 빈 앱 배포 |
| Phase 3~5 | 사장님·파트너 흐름 + 결제 동작 |
